# 项目上下文

## 项目架构

Workbench 是一个全栈个人工具箱网站：
- **后端**：Python 3.8+ / FastAPI / SQLite / JWT 认证
- **前端**：React 18+ / TypeScript / Vite / Tailwind CSS
- **部署**：本地优先，start.sh 一键启动前后端

### 核心机制
- **工具自动发现**：`backend/main.py:discover_tools()` 扫描 `backend/tools/` 目录，自动注册 router
- **工具规范**：每个工具需包含 `router.py`（FastAPI APIRouter）+ `meta.json`（元数据）
- **前端动态路由**：`App.tsx` 注册工具路由，`Home.tsx` 调用 `/api/tools` 展示工具列表

## 编码规范

- Python：Ruff 格式化，类型注解，Pydantic BaseModel
- TypeScript：strict mode，camelCase 变量，函数组件 + hooks
- CSS：Tailwind CSS，CSS 变量主题系统

## 依赖约束

- 后端：FastAPI + SQLAlchemy + PyJWT，不引入 Django/Flask
- 前端：React + React Router + Tailwind CSS，不引入 Redux/MUI
- 数据库：SQLite（本地优先），不引入 PostgreSQL/MySQL

## 测试策略

- 框架：pytest（后端）
- 结构：`tests/unit/backend/<tool_id>/` 镜像源码结构
- conftest.py：根级别 `tests/conftest.py` 放共享 fixtures
- 目标：核心逻辑覆盖率 ≥ 80%

## Git 规范

- 分支：`master`（当前），功能开发用 `feature/xxx`
- Commit：Conventional Commits（feat/fix/docs/refactor/test/chore）
- 每个功能完成后立即 commit + push
